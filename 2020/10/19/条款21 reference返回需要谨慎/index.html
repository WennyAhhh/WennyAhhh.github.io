条款21 reference返回需要谨慎

返回的对象如果是创建在栈中的，比如

const Rational& operator*(const Ration& lhs,const Ration& rhs){
	Ration result(...);		//构造函数，懒得写参数了。。。
	.....
	return result;
}
//客户端调用
Ration a(1,2);
Ration b(3,4);
Ration res=a*b;	//是错误的

以上代码会发生致命错误：在栈上实例化，在跳出作用域的时候调用析构函数，导致返回一个空引用。

如果在堆上创建，并且返回一个引用（当然不可能返回一个实例，如果返回一个实例的话就没地方delete）

const Ration& operator*(const Ration& lhs,const Ration& rhs){
	Ration *result=new Ration(...);
	return *result;
}

这样会导致一个错误，内存泄漏。很容易会忘记delete，就算很小心翼翼，也会在不经意间出错

Ration res=a*b*c;	//中间结果没有delete

如果采用static,如果清楚全局变量，就会理解这种错误

const Ration& operator*(const Ration& lhs,const Ration& rhs){
	static Ration result(....);
	...
	return result;
}
//这样由返回的是引用，所以==前后返回的都是“现值”，也就是说 这个if语句只会返回true；
if((a*b)==(c*d)){
	
}

所以，不要盲目返回reference，还是使用by-value吧。在C++11中，有了move和forward，可以解决调用拷贝函数和析构函数的烦恼。
