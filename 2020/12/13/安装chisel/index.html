rocket环境配置

概述

这个环境我安装了挺久了，分别在ubuntu和centos中安装。对于chisel和rocket-chip其实并没有改变多少。由于在rocket-chip的官方教程中有rocket-tools，我也没有多想就尝试安装，结果子模块会出现各种各样的问题，最开始由于渣渣校园网导致经常下载到一半断网，借了个教师账号才下载完成。编译中遇到各种稀奇古怪的编译问题，有些问题甚至google不到，最终都是通过下载官方github编译运行的。

后来发现我的任务好像只是运行rocket-chip

这里的chisel-tutorial的build.sbt需要修改。

libraryDependencies的各种组件可能因为版本原因，需要升级。可以在https://mvnrepository.com/artifact/edu.berkeley.cs中寻找最新的模块

// build.sbt
def scalacOptionsVersion(scalaVersion: String): Seq[String] = {
  Seq() ++ {
    // If we're building with Scala > 2.11, enable the compile option
    //  switch to support our anonymous Bundle definitions:
    //  https://github.com/scala/bug/issues/10047
    CrossVersion.partialVersion(scalaVersion) match {
      case Some((2, scalaMajor: Long)) if scalaMajor < 12 => Seq()
      case _ => Seq("-Xsource:2.11")
    }
  }
}
 
def javacOptionsVersion(scalaVersion: String): Seq[String] = {
  Seq() ++ {
    // Scala 2.12 requires Java 8. We continue to generate
    //  Java 7 compatible code for Scala 2.11
    //  for compatibility with old clients.
    CrossVersion.partialVersion(scalaVersion) match {
      case Some((2, scalaMajor: Long)) if scalaMajor < 12 =>
        Seq("-source", "1.7", "-target", "1.7")
      case _ =>
        Seq("-source", "1.8", "-target", "1.8")
    }
  }
}
 
name := "MyChisel"
version := "3.2-SNAPSHOT"
scalaVersion := "2.12.6"
crossScalaVersions := Seq("2.11.12", "2.12.4")
 
resolvers += "My Maven" at "https://raw.githubusercontent.com/sequencer/m2_repository/master"
// bug fix from https://github.com/freechipsproject/chisel3/wiki/release-notes-17-09-14
scalacOptions ++= Seq("-Xsource:2.11")
 
libraryDependencies += "edu.berkeley.cs" %% "chisel3" % "3.4.0"
libraryDependencies += "edu.berkeley.cs" %% "chisel-iotesters" % "1.5.0"
libraryDependencies += "edu.berkeley.cs" %% "chisel-testers2" % "0.2.2"
libraryDependencies += "edu.berkeley.cs" %% "rocketchip" % "1.2.2"
 
scalacOptions ++= scalacOptionsVersion(scalaVersion.value)
javacOptions ++= javacOptionsVersion(scalaVersion.value)

Rocket-tools

本安(xin)装(lu)教(li)程(cheng)不保证时效性。以官方教程为主。注意区分路径，还有google的蛋疼翻译。

官网的环境依赖，但是在centos有些不可以用，有些版本太低了。不建议没有代理工具安装，即使可以利用码云，zygxsq.cn。因为他包括很多子模块，地址也都是github。当然要是有心情的话可以慢慢修改.gitmoudle文件，但是限于子模块实在太多，这个项目的子模块嵌套了三层，不建议这样做。

在ubuntu上有时候automake版本够了仍然报错

gcc版本要高

# 升级gcc
yum install -y centos-release-scl
yum install -y devtoolset-6-gcc devtoolset-8-gcc-c++
source /opt/rh/devtoolset-8/enable
# 官方依赖，有些在centos官方库没有，有些版本太低。
sudo yum -y install autoconf automake autotools-dev curl libmpc-dev libmpfr-dev libgmp-dev gawk build-essential bison flex texinfo gperf libtool patchutils bc zlib1g-dev libexpat-dev
# automake官方库版本低，查到的资料里还有autoconf，顺便也升级了一下
sudo rpm -e --nodeps autoconf-2.63 
wget http://ftp.gnu.org/gnu/autoconf/autoconf-2.70.tar.gz
tar zxvf autoconf-2.70.tar.gz
cd autoconf-2.70
sudo ./configure --prefix=/usr
sudo make && make install

make && make install

前面只是最简单的前置工作。

后续工作中需要注意的是路径地址

# 编译之后保存的地址
export RISCV=/home/chaikaichen/to/install/riscv/toolchain
# 保存在运行在riscv上的工具，比如gcc。
export PATH=$RISCV/bin:$PATH
# 设置环境变量，注意路径
export RISCV=/home/chaikaichen/to/install/riscv/toolchain

# rocket工具链
https://github.com/chipsalliance/rocket-tools
# 下载子模块。这个子模块相对而言很大，其中有个riscv-qemu，官方master已经关了，所以没有git https了，只有git://。这个url导致代理直接失效，70k/s已经很幸福了
# 当遇到这个子模块，玄学一点，多试一试 git submodule update --init --recursive ，诚意够了，速度就上300K/s。
git submodule update --init --recursive
# N是几个核的意思，取决于cpu或者虚拟机设置。手冷可以多开一点取暖
export MAKEFLAGS="$MAKEFLAGS -jN"
# 自带的riscv-gnu-toolchain版本有些低，make的时候出现error。需要重下
rm -rf riscv-gnu-toolchain
git clone https://github.com/riscv/riscv-gnu-toolchain
cd riscv-gnu-toolchain
git submodule update --init --recursive
./configure --prefix=$RISCV
make
export PATH=$RISCV/bin:$PATH
# 会有各种各样的错误，我差点给跪下了
# 需要编译很长时间，还会有各种error和fatal。看到warning就谢天谢地吧
cd ..
./build.sh

我在编译运行时，有时候运行会错误，导致后续的无法编译。例如我在运行过程中riscv-tests无法编译。可能不是环境问题不知道的统统是版本问题.直接删除这个子模块，在官方github中下载，编译，例如riscv-tests。但是此方法一定要先编译运行riscv-gnu-toolchain因为要用到gcc

git clone https://github.com/riscv/riscv-tests
cd riscv-tests
git submodule update --init --recursive
autoconf
./configure --prefix=$RISCV/target
make
make install
