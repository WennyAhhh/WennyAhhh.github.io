chisel测试问题

chisel测试问题

查了一下相关的资料（第三章），找到了这个图。

关于函数传参中可能出现的疑惑，我也在其他资料中找到了。的确是柯里化，只是因为scala的函数字面量有些晦涩

代码中用的是chisel3.iotesters这个模块，这个模块是chisel的测试模块（官方api资料），可以搭配verilator仿真，即使我加入了参数，使用了verilator，测试还是调用chisel的这个模块而已，所以用于测试的模块到底来说都是同一个，都是依靠以下TesterSimple进行输出测试信息的，而这个类是使用chisel3.iotesters所需要的参数的组成部分，和有没有参数是没有关系的。

这个是测试模块，需要继承测试的类。

chisel3.iotesters.Driver需要创建第二参数列表，也就是一个测试类，创建了这个测试类才能进行测试。如果需要生成其他文件就要使用execute函数，比如vcd。就可以再使用一个至于参数，不仅仅只有verilator这一个，我还找到其他参数，其中就包括以下这个，这个参数就直接生成vcd文件而没有cpp等文件了。如果没有使用参数，但是使用了chisel3.iotesters.Driver，有测试信息，并且但是没有vcd文件。

如果不加这个模块就不会进行测试了。就是另外一个方向了（但是查看文档这个execute函数被弃用了）。

这样就没有测试信息提示了，并且生成Verilog文件。

代码：

import chisel3._
import chisel3.Driver
import chisel3.iotesters.PeekPokeTester
import org.scalatest._

class DeviceUnderTest extends Module{
    val io = IO(new Bundle{
        val a = Input(UInt (4.W))
        val b = Input(UInt (4.W))
        val out = Output(UInt(2.W))
    })
    io.out := io.a & io.b
}

class TesterSimple(dut : DeviceUnderTest) extends PeekPokeTester(dut){
    poke(dut.io.a, 3.U)
    poke(dut.io.b, 1.U)
    step(1000)
    //println("Result is: "+peek(dut.io.out).toString)
    expect(dut.io.out,1)
    poke(dut.io.a, 2.U)
    poke(dut.io.b, 0.U)
    step(1000)
    //println("Result is: "+peek(dut.io.out).toString)
    expect(dut.io.out,0)
}
//直接生产vcd文件
object TesterSimple extends App{
    chisel3.iotesters.Driver.execute(Array("--generate-vcd-output", "on"),() => new DeviceUnderTest()){ 
        c => new TesterSimple(c)
    }
}
//需要自行添加参数
object TesterSimple extends App{
    chisel3.iotesters.Driver.execute(args,() => new DeviceUnderTest()){ 
        c => new TesterSimple(c)
    }
}

//chisel验证，不生成文件，有测试信息
object TesterSimple extends App{
    chisel3.iotesters.Driver(() => new DeviceUnderTest()){ 
        c => new TesterSimple(c)
    }
}

//生产verilog文件，但是没有测试信息
object TesterSimple extends App {
  chisel3.Driver.execute(args, () => new DeviceUnderTest())
}
